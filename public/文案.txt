BRIX Framework
下一代插件化企业应用平台

Brix Framework 是一个 插件驱动（Plugin-Driven）的企业级平台架构，用于构建可扩展、可组合的行业应用系统。

通过 统一能力契约（Capability Contract）+ 插件运行时（Plugin Runtime），Brix 将复杂的企业系统拆解为 可独立开发、部署、升级的业务插件，使企业应用从传统的单体系统，演进为 可组合的软件平台（Composable Platform）。

Brix 支持三种运行模式：

Product Mode
构建完整的独立平台产品

Embedded Mode
将业务能力以插件形式嵌入客户系统

SDK & API Mode
开放平台能力，支持合作伙伴接入

这一架构使 Brix 同时具备：

产品平台 + 行业解决方案 + 开放生态平台 三种能力。

Plugin-Driven Platform
用插件构建企业系统

传统企业系统通常具有以下问题：

系统复杂度随着业务增长指数级上升

模块耦合严重，升级困难

不同客户需求难以快速定制

系统演进成本极高

Brix 通过 插件化能力架构，将企业系统拆分为 独立业务能力插件（Business Plugins）：

例如：

Patient

Doctor

Appointment

Payment

Insurance

Analytics

每个插件都是一个 完整的业务能力单元，具备：

独立开发

独立部署

独立升级

生命周期管理

企业应用不再是一个巨大的系统，而是 由能力插件组合而成的平台。

Brix 3.0.7 技术架构优势

Brix 3.0.7 在原有架构基础上，引入了 运行时能力层（Runtime Layer）与能力契约体系（Capability Contract），使平台具备更高的扩展能力和长期演进能力。

核心技术架构由五个层级组成：

1 Host Layer（平台宿主层）

Host Layer 负责 插件运行环境与系统入口。

包括：

Standalone Host（独立平台）

Embedded Host（嵌入式平台）

API Gateway（开放接口层）

这一层实现了：

平台产品化 + 客户系统嵌入 + 开放平台

三种商业模式。

2 Capability Contract（能力契约层）

这是 Brix 3.0.7 最核心的设计。

Capability Contract 定义了插件之间 统一的运行时接口标准：

包括：

EventBus

StateStore

AuthContext

Lifecycle

Observability

Config

这一层相当于 插件操作系统（Plugin OS）。

所有插件只需要遵循 Capability Contract，即可在不同 Host 中运行。

这使得 Brix 具备：

跨系统运行能力 + 插件生态能力

3 Business Plugins（业务插件层）

企业业务能力以插件形式存在，例如：

Patient

Doctor

Appointment

Payment

Insurance

Analytics

插件具备：

UI Module（React组件）

Service Module（业务逻辑）

Manifest（插件描述）

并支持完整生命周期：

Install → Activate → Update → Disable → Remove

这一机制让企业系统像 应用商店一样管理功能模块。

4 Frontend Runtime（前端运行时层）

Brix 3.0.7 引入了 Frontend Runtime Layer，用于统一前端能力。

包括：

React Runtime

Router

State Management

UI Library

这一层使前端插件具备：

模块隔离

动态加载

统一运行环境

从而实现 前端插件化架构。

5 Infrastructure（平台基础设施）

平台底层采用现代云原生架构：

PostgreSQL

Redis

Kafka

OpenTelemetry

Kubernetes

并通过 Service Adapters 进行统一接入。

这一设计使 Brix 具备：

高可扩展性 + 云原生能力 + 高并发支持

Full-Stack Plugin Architecture
前后端对称的插件架构

Brix 采用 Full-Stack Symmetry Architecture：

前端与后端均以插件形式存在。

插件包含：

UI Module
Service Module
Manifest

并通过 统一能力契约运行在 Host Runtime 之上。

这一架构带来的优势：

前后端统一开发模式

插件独立部署

系统扩展成本极低

支持大型平台长期演进
